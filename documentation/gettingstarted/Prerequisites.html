Prerequisites

This article provides you with the information required before you start importing a LibGDX project.

LibGDX projects are built and managed with the Gradle build tool. This allows you to develop in almost any environment that takes your fancy, even if you are working in a team.

The 3 major java IDEs, IDEA/Android Studio, Eclipse and Netbeans are all supported, just make sure you have set them up correctly.

Target platform setup

If you would like to target iOS, please see below for additional setup information.

iOS

If you want to target iOS, you can use one of the iOS backends.

  • Robovm OSS

    We have RoboVM OSS, the OSS fork of RoboVM maintained by us. You must have XCode 7+ installed, and yes you must have a Mac to deploy and test on device/simulators.

  • Multi OS Engine

    You can also use the Multi OS Engine, which will let you develop on Windows and on Mac. On Mac you must have XCode 7+ installed, and an account to develop on the cloud.

Setting up Intellij IDEA/Android Studio

Minimum:

  • Java Development Kit 7+ (Must be JDK)
  • Intellij IDEA 2016+, community edition is sufficient
  • Android Studio, 2.2+

Android:

  • Android SDK

iOS:

  • RoboVM OSS Intellij plugin (if RobovmOSS)
  • Multi OS Engine plugin (if Multi OS Engine)

Setting up Eclipse

Minimum:

  • Java Development Kit 7+ (Must be JDK)
  • Eclipse Mars+

Android:

  • Android SDK
  • ADT Tools

iOS:

  • RobovmOSS Eclipse plugin
  • Multi OS Engine plugin
